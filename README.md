# ACG ProPack

ACG ProPack is a warehouse and shipment management web application made for companies that need to organize packages, track warehouse receipts, manage storage locations, and handle shipping in one system. The project was created as a senior project to show how software can improve day to day business operations by making inventory and billing processes easier to manage.

The system allows a company to create and manage its own workspace, add team members, organize warehouse activity, and keep track of package movement from receiving to shipping. It also includes billing features so the company can manage plans, payments, and subscription related actions in one place. The goal of the project is to give businesses a more structured and professional way to manage logistics work.

## Main Features

Warehouse receipt and package tracking

Inventory and storage location management

Shipment creation and shipment tracking

Company account and member management

Role based access for company users

Billing, payment methods, and subscription management

Frontend and backend separated for easier development

## Project Structure

The project is divided into two main parts.

`frontend` contains the user interface and pages that users interact with

`backend` contains the API, database models, and business logic

Each folder has its own README file with instructions on how to install dependencies and run that part of the project.

## Project Purpose

The purpose of ACG ProPack is to solve common problems in warehouse and logistics work. Instead of using disconnected spreadsheets or manual notes, this system provides one place where users can manage operations more clearly. It also gives a more realistic example of how a business application can be designed with both technical and user needs in mind.

This project was developed as part of a senior project and represents work in planning, software design, implementation, and testing.
